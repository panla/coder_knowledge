# MongoDB 复制

复制是一种在多个服务器上同步数据的过程。通过在不同的数据库服务器上实现多个数据副本，复制能够实现数据冗余，提高数据的可用性

## 运作方式

MongoDB 使用副本集（replica set）来实现复制操作。副本集是一组托管同一数据集的 mongod 对象。在副本集中，主节点负责接收写入操作。所有其他的实例（从节点）则通过执行主节点的操作来拥有同样的数据集。副本集中只有一个主节点。

- 副本集有2个或多个节点
- 副本集只有一个主节点
- 从节点所有数据都是从主节点复制来的
- 推选新主节点

## 副本集特点

- 具有 N 个节点的集群
- 任何节点都可能成为主节点
- 由主节点负责写
- 自动故障转移与故障恢复
- 重新推举主节点
